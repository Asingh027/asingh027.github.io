This project focuses on developing an automated stock trading algorithm that buys and sells shares of stock daily based on recommendations of trading signals. Technical indicators such as Moving Average Convergence Divergence (MACD), Relative Strength Index (RSI), and Bollinger Bands are implemented to evaluate a stock price's current trend.

The Alpaca API is utilized to check current positions and push buy/sell orders based on the evaluator's recommendation. This allows for real-time trading decisions to be made based on the algorithm's analysis of the technical indicators.

The project showcases strong leadership qualities by utilizing state-of-the-art technical indicators to inform trading decisions and automate the process, resulting in more efficient and effective trading strategies.

Student Registration Tool

The Student Registration Tool project involved designing an Entity-Relationship Diagram and a Normalized Relational Schema to optimize the management of student registration data. The team utilized MySQL commands to implement the schema and locally hosted the database.

Using Python, the team connected to and queried the MySQL database for information about students, courses, and professors. This enabled them to develop a powerful and efficient tool for managing student registration data.

To ensure that users could easily navigate the tool and take full advantage of its features, the team generated a comprehensive user guide that provided in-depth instructions on how to use the tool effectively.

The Student Registration Tool project showcased the team's expertise in database management, SQL programming, and Python. Their ability to design and implement a normalized relational schema using MySQL highlights their proficiency in optimizing data storage and retrieval. Moreover, their attention to detail in creating a user-friendly guide underscores their commitment to delivering high-quality solutions that meet user needs.

Twitter Sentiment Analysis

The Sentiment Analysis project involved training a Logistic Regression Machine Learning (ML) algorithm on a large dataset comprising 1,600,000 rows. The objective of the project was to classify the sentiment of tweets as either positive (1) or negative (0).

The team utilized the Twitter API to fetch relevant tweets, which were then classified using Spark MLib. This enabled the team to perform sentiment analysis on the tweets and classify them with high accuracy.

To store the fetched tweets' data and classification results, the team utilized an external database. Additionally, the team evaluated the accuracy of their model via ROC curve analysis, which enabled them to measure the model's performance and fine-tune it for optimal results.

The project showcased the team's expertise in machine learning, data analysis, and database management. Their ability to utilize cutting-edge technologies such as Spark MLib and the Twitter API highlights their proficiency in leveraging advanced tools and techniques to solve complex problems.

Reddit Score Regressor

The Reddit Score Regressor project employed Natural Language Processing (NLP) methods, classification and regression models to predict the upvote score of "Florida Man" articles on Reddit. The team achieved an impressive accuracy rating of 97% by utilizing advanced machine learning techniques.

To preprocess the data, the team leveraged Tokenizer, a tool from Keras, which converted text to numerical values for training. This enabled the development of a robust regression model that utilized cutting-edge deep learning techniques, including word embeddings and fully connected layers. These techniques allowed the team to extract features from the title headlines of "Florida Man" articles and accurately predict the Reddit score with precision.

The project showcased the team's technical expertise in machine learning and demonstrated their proficiency in utilizing advanced ML methods. The success of the project is a testament to the team's collaborative approach, data-driven decision-making, and innovative problem-solving skills.

Machine-Learning Powered Oil & Gas Delivery Route Optimization

This project focuses on optimizing oil delivery routes using data clustering methods. The team utilizes unsupervised learning techniques such as k-means clustering to determine the optimal delivery routes based on factors such as distance, fuel consumption, and delivery time.

A scalable and efficient AWS pipeline is developed to automate data processing and analysis, including data cleaning and feature engineering. Real-time data such as traffic and weather conditions are incorporated to ensure the optimized routes remain efficient and effective over time.

Insights and recommendations from the analysis are then summarized in an interactive dashboard using tools such as Tableau and Power BI. This allows leadership to make informed decisions based on the data.

Overall, this project showcases strong leadership qualities by using data clustering methods to optimize oil delivery routes, enabling efficient use of resources and real-time optimization.
